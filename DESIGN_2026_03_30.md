## 4. データフロー

### 【管理者】
```
① admin.html（GAS デプロイ URL で開く）
   ↓
② クラス設定を追加（例：2A, 40人）
   ↓
③ admin_gas.gs: saveClassConfig()
   ↓
④ スプレッドシート「クラス設定」シートに保存
   ↓
⑤ config.js 自動生成
   ↓
⑥ GitHub Pages の quiz_prototype_v2.html が新設定を反映
```

### 【生徒】クイズ挑戦
```
① quiz_prototype_v2.html（GitHub Pages から開く）
   ↓
② config.js からクラス一覧（YEAR_CONFIG）を読み込み
   ↓
③ クラス・番号を選択 → 「始める」ボタンクリック
   ↓
④ quiz_api.gs: getActivePeriod() で提出期間確認
   → 期間情報を sessionStorage に保存
   ↓
⑤ quiz_api.gs: getQuestions() でクイズ3問取得
   ↓
⑥ 回答・採点
   ↓
⑦ 3問全問正解 → 「レポート提出」ボタン表示
```

### 【生徒】レポート提出
```
① 「レポート提出」ボタンクリック → submission.html へ遷移
   ↓
② submission.html で sessionStorage から期間情報を取得
   ↓
③ ローカルで期間チェック
   ├─ 期間内：アップロード許可
   └─ 期間外：エラーメッセージ表示「提出期間外です」
   ↓
④ 写真/PDF をアップロード
   ↓
⑤ submission.html: POST で submission_gas.gs へ送信
   ↓
⑥ submission_gas.gs: 念のため再度期間チェック（サーバー側検証）
   ↓
⑦ Google Drive のフォルダに保存
   ↓
⑧ OCR 処理（Gemini API → Claude API のフォールバック）
   ↓
⑨ スプレッドシート「OCR結果」に記録
   ↓
⑩ 提出完了メッセージ表示
```
